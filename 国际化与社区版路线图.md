# 国际化与社区版路线图

> 2026-07-22 立项：用户反馈基本验证完毕，进入"主场转向海外"落地阶段。四条工作线：① 社区版 ② 英文默认+KJV ③ 多语言 ④ Cloudflare 上线。

## 核心架构决策：中文缩写 = 语言无关内部 ID

章键（`拿4`）、节键（`拿4:2`）只是 ID——**beads 珠库（14.2 万对）、URL 格式、本机存档、全部串珠逻辑零改动**。每种语言只是一层皮：

```
语言层 = verses-<lang>-N.js（节文本分片） + books-<lang>.js（书名表）
       + UI 字符串字典 + 找线锚词算法（按语系） + 叙事文案
```

KJV 是 TSK 的母语：珠对本就按 KJV 编，英文版反而是"回家"。

## W1 英文版（默认语言）

1. ✅ **KJV 数据层（2026-07-22 完成）**：`data/verses_en.py` → verses-en-1/2/3.js（31,100 节，键同 CUV：`拿4:2` → KJV 文本，去 `{}` 补字标记）+ books-en.js（BOOKFULL_EN/BOOKSHORT_EN）。创1:1/拿4:2/出34:6/诗23:1/约3:16/启22:21 抽查通过。
2. UI 字典化：四页全部字符串收进 STRINGS[lang]；语言判定：默认 **en**，`?lang=zh` / localStorage / 浏览器语言覆盖；分享文案、PWA manifest 同步双语。
3. 英文找线算法：词级 tokenize + KJV 停用词（the/and/of/unto/shall/thee/hath…）+ 简单词形归一（-eth/-est/-s 后缀），替换中文 n-gram 逻辑（zh 保留原算法）。
4. 版式适配：汉字 letter-spacing 排版对 en 关闭；Georgia 本就是英文衬线，绘本皮肤圆体族补英文 rounded 回退。
5. 叙事英文化：沙漠修士文案、深度名（浅滩珠→the Shallows / 深海珠→the Deep 等）、呼吸引导语。
6. 节号差异兜底：CUV/KJV 约 30 处版本节号错位（诗篇题记并节、约叁 14/15 等）——英文侧查不到键时回退上一节，不报错。

## W3 Cloudflare 上线（阻塞在用户侧账号）

**需要用户做的（一次性，约 20 分钟）**：
1. 注册 Cloudflare 账号（cloudflare.com，免费）
2. 域名：Cloudflare Registrar 直购（成本价 ~$10/yr）或已有域名接入
3. 给我一个 Pages 写权限的 API Token（My Profile → API Tokens），或在控制台建好 Pages 项目后走 wrangler 登录

**我这边就绪后**：deploy.ps1 加双目标——dist/ 原样 `wrangler pages deploy`，CloudBase 继续做国内镜像；主域指 Cloudflare，站内分享链接换主域。

## W4 社区版（阻塞在用户侧 Supabase 账号）

- **栈**：Supabase 免费档起步（Auth + Postgres + RLS），前端仍是静态站直连。
- **认证**：邮箱 magic link + Google 登录（海外用户习惯）；非匿名才可写。
- **表设计（草案）**：
  - `profiles`：id / 显示名 / 等级 / 创建时间
  - `strands`：user_id / pair_key（章对键） / anchors / anchorsB / reason / lang / 时间——云端串档
  - `amens`：strand_id / user_id——「造就了我」
  - `community_beads`：创建者 / a_ref / b_ref / 审议状态 / 月配额记账——自创珠（P2c 才开）
  - `reports`：举报/纠错
- **分期**：P2a 注册 + 本机存档一键并入 + 云同步 → P2b 众人的注脚（读他人理由）+ 造就了我 → P2c 串珠人等级 + 自创珠 + 社区审议。
- 铁则不变：平台不创作解释；社区集与 TSK 集永远分层；LLM 只做翻译。

## 执行顺序

**W1 英文版 →（W3 账号一到位就双部署）→ W2 先上 1-2 门 → W4 社区版**

W1 是其余一切的地基（海外上线前必须默认英文）；W4 最后动，因为它需要真实海外流量来验证。
