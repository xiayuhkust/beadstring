# 可获取题目来源盘点

> 2026-07-21 · 产品重心已定：**沉浸式"短读经 + 做题"**——以题目为单位，读与题相关的原文短段（3-8 节），随后作答。整章通读计划不做（市场已饱和）。
> 铁律：凡有正确答案的题必须带 `source`；引文逐字对照和合本。

---

## A 级 · 结构化 + 开放授权 + 可机读（题库主力）

### 1. unfoldingWord® Translation Questions（uW tQ）⭐ 最大发现
- **是什么**：为圣经翻译检查而作的**全圣经逐章、逐段理解题**（问答对形式），每题天然锚定一个短段——与我们"题目牵引短读经"的单元完全同构
- **授权**：CC BY-SA 4.0（可翻译、可改编，需署名+同协议共享）
- **获取**：[git.door43.org/unfoldingWord/en_tq](https://git.door43.org/unfoldingWord/en_tq)，TSV 格式，按书卷成文件，可 git clone
- **用法**：英文问答对 → 中译 → 改写为选择题（正确答案来自其 answer 字段，干扰项按"常见误读"人工设计）
- **出处标注**：`unfoldingWord® Translation Questions v88 · 拿 4`
- **体量**：覆盖全本 66 卷，估计数千问答对

### 2. OpenBible.info 串珠数据（TSK 现代化整理）
- **是什么**：约 34 万条经文交叉引用，主体源自公有领域的 Treasury of Scripture Knowledge，带用户投票权重（可取高票=最公认的联结）
- **授权**：CC Attribution；**2MB zip 直接下载**：[openbible.info/labs/cross-references](https://www.openbible.info/labs/cross-references/)
- **用法**：批量生成联结题（"本段与哪处经文呼应？"），干扰项取其他书卷的不相关段
- **出处标注**：`TSK / OpenBible.info cross-references`

### 3. crossreferences.org 短语级串珠
- 约 30 万条引用 + 6.4 万个**短语级锚点**（精确到句中短语，比章节级更细），CC BY-SA 4.0，GitHub 实时数据：[crossreferences.org](https://crossreferences.org/project/roadmap/)
- 用法：高级联结题（"『影儿遮盖他的头』让人想起哪里？"）

### 4. 和合本结构化文本（公有领域）
- [xieyao04/bible](https://github.com/xieyao04/bible) 等 GitHub 数据库
- 三重角色：短读经呈现本体 / 观察题事实校验源 / 引文机器比对基准

## B 级 · 公有领域经典（人工提炼，出处清晰）

### 5. 要理问答（题目形态的现成神学内容）
- 威斯敏斯特小要理问答（107 问）、海德堡要理问答（129 问）、路德小问答——原文公有领域
- **关键结构**：每问附证明经文（proof texts）→ 可反向映射到经文段落，生成解释题
- 出处标注：`威斯敏斯特小要理问答 第 21 问`
- ⚠ 中译本需用公版旧译或自译（现代译本有版权）

### 6. 公有领域注释（解释题的依据文献）
- Matthew Henry、Jamieson-Fausset-Brown、Barnes' Notes、司布真《大卫宝库》（诗篇）——英文公有领域
- 用法：解释题正确答案的文献依据；出处标注注释家名 + 卷章

## 补：串珠数据低分区构成（2026-07-21 实测）

分布：100+ 票仅 1,327 条；**87% 的珠在 0-9 票区**；负分 1,248 条（0.36%）。低分区四种成分：

1. **冷门宝石（多数）**：完全成立但无人路过——拿4:2↔王上19:4（两个在植物下求死的先知，2票）、哀4:10↔申28:53（预言应验，1票）、何2:14↔创34:3/士19:3（希伯来成语"对她的心说话"同款，2票）
2. **公式联结**：仅共享套语（"我不愿意你们不知道"式），灵修价值低
3. **错误**：负分区≈社区勘误表（弗6:17↔撒上17:58，-86，疑 TSK 手误），直接排除
4. **被大众错杀**：路15浪子↔太21两个儿子比喻 -19——票数是英语大众口味，非真理

**策略**：只排除负分；相对排名用于排期不用于筛除；低分区做等级内容（"深水珠"高等级开采）；社区审议兼具"为老珠重新定价"使命——中文社区可赎回被错杀的珠。

**负分区分层政策（2026-07-21 修订，取代"一律排除"）**：负分 1,248 条中 -1~-3 占 1,067。三层处理——① 重负分（≤-10，仅 25 条）+ 机器可识别的公式句（章内格式句互指、标题句、套语）：排除；② **轻负分（-1~-6，约 1,190 条）：进"争议区"不排除**——实测藏有高价值争议珠：创1:3↔约11:43（-5，「要有光就有了光」↔「拉撒路出来！那死人就出来了」——话语行事的基督论，踩它的人看词面，爱它的人看神学）、路13:24↔约13:33（-1，两个"不能"）；③ 社区审议第三使命：**翻案**——高等级串珠人陪审争议珠（含被错杀的重负分如浪子↔两个儿子 -19）。

**双轴模型（2026-07-21 澄清）**：票数=「热度」（被看见的程度，流量支配——证据：拿4:2 逐字引用出34:6 仅 30 票，诗23:1↔腓4:19 主题相似却 440 票），不是「强度」。强度独立分档、可客观测量：① 明文引用级（新约引旧约对照表可枚举）＞② 同词/同成语级（共词算法自动测）＞③ 传统收录级（TSK 编者主题判断）；负分=社区勘误，排除。低正分≈"没人看见"，非"联系弱"——社区真不认可会打负分。产品数据结构中热度与强度为两个独立字段；社区投票只影响热度轴。

## 单元结构 v2（题目牵引的短读经）

```js
{ id: "JON.4.q1",
  passage: "拿 4:1-4",            // 短段，3-8 节
  verses: ["约拿却大大不悦，且甚发怒……"],   // 和合本原文，逐字
  question: { tier: "观察", q: "…", opts: […],
              source: "uW tQ v88 · 拿 4 / 经文本身",
              explain: "…（拿 4:2）" } }
```

沉浸流程：入场（场景+呼吸，可短至 20 秒）→ **原文短段逐句浮现** → 题目 → 解析/延伸 → 点亮。单次 2-3 分钟，一天可自由做多个。

## 流水线 v2

1. uW tQ TSV 拉取 → 中译 → 改写选择题（source 自动继承）
2. OpenBible 串珠高票条目 → 联结题模板生成（source 自动继承）
3. 要理问答 proof-text 反查 → 解释题（source = 问号）
4. 开放题人工撰写（无需正确答案来源，延伸句注明传统出处）
5. 机器校验引文 ↔ 和合本 DB → 人工/牧者终审
