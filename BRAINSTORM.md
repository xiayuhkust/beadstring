# 灵修网站 · 头脑风暴

> 目标：用游戏行业的注意力设计，帮助基督徒提高灵修专注度。
> 原则：**恩典而非律法** —— 一切激励机制不制造焦虑、不惩罚中断。
> 技术基线：纯静态网页，零构建，CloudBase 部署分享（可复用 vfx-lab 的特效原子）。

---

## 一、氛围场景（沉浸感）

- **烛光祷告室** ⭐ 首选原型
  - 粒子蜡烛 + 全黑背景，点燃即开始祷告
  - 蜡烛燃烧本身就是计时器（5/10/15 分钟），比数字倒计时安静
  - 切走页面火苗变暗（温和提示），但不会熄灭
- **经典圣经场景**：可安歇的水边（诗 23）、旷野、夜空星辰（诗 8）
- **昼夜联动**：清晨=晨光色调，深夜=烛光色调
- **教会年历联动**：将临期紫色、复活节白金色

## 二、节奏引导（放慢速度）

- **呼吸圆环**：光圈收缩扩张引导呼吸，吸气浮现上半句经文，呼气浮现下半句（Lectio Divina 数字化）
- **经文逐词淡入**：像过场字幕一样逐词浮现，强制慢读——快读是默想的天敌
- **生成式环境音**：Web Audio 合成雨声/风声/远钟，无需大音频文件

## 三、激励机制（克制！）

- ✅ "点亮的日子"日历，中断不清零、不惩罚
- ✅ Forest 式蜡烛：专注期间燃烧，分心变暗
- ❌ 排行榜、成就轰炸、签到弹窗、streak 断裂焦虑

## 四、碎片化读经（待深入讨论）

**痛点**：现有读经计划动辄 1-2 年线性读完，门槛高、挫败感强、掉队即弃。

**核心思路：从"进度条"改成"马赛克墙"**

- 圣经共 1189 章 → 呈现为一面**点亮地图**（按书卷分区的格子墙）
- 每读完一个碎片，点亮对应格子；格子**永远亮着**，只累积不倒退
- 不规定顺序、不规定速度——今天读哪块都算数，掉队这个概念不存在

**碎片单元设计（3-5 分钟一个）**

1. **入场 10 秒**：烛光/呼吸引导，快速切换状态
2. **上下文胶囊 30 秒**：两三句话交代"这段发生在什么背景下"，解决碎片化最大的问题——断章取义和迷失感
3. **正文 2-3 分钟**：逐段浮现式阅读
4. **回应 30 秒**：一句默想问题或一句祷告

**任务发放方式（候选）**

- **每日抽取**：每天发一个碎片（可以是编辑精选的"黄金段落"路线）
- **主题线**：恩典线 / 应许线 / 人物线（大卫的一生）/ 祷告线——像游戏支线任务，短线 7-21 个碎片可完结，给完结感
- **回响机制**：偶尔重发已读过的碎片做"再访"（间隔重复），旧格子闪一下光

**关键取舍**

- 碎片化 ≠ 放弃通读：马赛克墙最终仍指向全本覆盖，只是路径自由
- 完结感靠"短主题线"提供（7 天可完成一条线），累积感靠"墙"提供

### 完整周期 ≠ 通读全本（三层同心环）

**争议点**：给章节分"重要程度"是神学雷区。
**化解框架**：不按重要性排名，按**叙事功能**组织；先例是教会两千年的主日经课集（Lectionary）——三年循环也只覆盖精选段落，从无"不完整"之讥。

- **第一环 · 故事弧线（约 90 碎片，约 3 个月）**：创造→堕落→立约→出埃及→王国→被掳→基督→教会→新天新地。走完 = 完整经历一次救恩叙事弧线，这就是"完整周期"奖励的名义
- **第二环 · 扩展环（约 365 碎片，一年）**：加入诗篇、智慧书、先知书精华
- **第三环 · 全本环（1189 章）**：马赛克墙终极形态，硬核用户向

**用"深度"代替"取舍"**：家谱/律法细则不跳过，而是做成"俯瞰卡"——一个碎片的上下文胶囊概括多章。墙上每格最终都能亮：深读金色格，俯瞰银色格。没有章节被宣判不重要，但时间花在刀刃上。

## 五、Pools 类氛围场景 × 沙漠灵修

（"核池" = Pools 类纯场景无战斗氛围游戏 / 利敏空间步行模拟）

**心理机制**：敬畏感（awe）——巨大、空旷、超身体尺度的空间触发谦卑与超越感。大教堂建筑学与沙漠教父选择旷野是同一原理。

**候选场景（全部有圣经原型）**

- 旷野/沙漠 —— 沙漠教父、耶稣四十天受试探；极远地平线 + 风声
- 何烈山的微小声音（王上 19）⭐ —— 烈风→地震→火的音效层层过去，归于近乎静默的微声；整个场景 = 一次听觉灵修
- 可安歇的水边（诗 23）—— 直接对上 Pools 水面美学
- 客西马尼深夜橄榄园 / 清晨空坟墓 / 平静风浪后的海面

**与主题线合流**：场景跟着叙事线走——出埃及线的碎片发生在红海边、旷野、西奈山脚；大卫线在山洞与旷野。空间是叙事的容器，不是背景板；这是 Pools 类游戏没有而我们有的：空间有故事授权。

## 六、场景变化系统（每章一景的工程化）

**核心原则：场景不是美术资产，是参数包。** 引擎只有少数"原子"（天空、地形、水面、植被、粒子、音频配方），每个场景 = 一份 JSON 预设。

**三层变化模型**

1. **L1 生态预设（biome）**：调色板（天顶/地平线/太阳/地面/雾 5 色）+ 地形函数参数（振幅/频率/走廊宽度）+ 道具表 + 音频配方。旷野/水边/橄榄园/海面 ≈ 6-8 个生态就能覆盖大部分叙事线。
2. **L2 状态调制**：昼夜时刻、天气、教会年历——只对 L1 的调色板做插值，不动几何。同一片旷野，黎明和深夜是两种灵修。
3. **L3 种子微变化**：`seed = 章节编号`，决定岩石位置、沙丘相位、星空布局。确定性带来**地点记忆**：重访同一章经文 = 回到同一个地方——这本身就有灵修意义。

**小星球呈现（messenger 式）**：~~两条路线~~ **已裁决（2026-07-20）：弯曲世界不是刚需，砍掉**。资源集中在"每章场景变化 + 氛围合拍"。

**引擎落地（2026-07-20）**：`scene-lab/` 已实现预设驱动引擎——`engine.js`（原子：天空/地形/水面/道具/粒子/音频/行走/流程）+ `presets.js`（场景=参数包）。已有两个生态：旷野（何 2:14）、水边（诗 23）。切换场景 = `?scene=` 参数，新增场景 = 加一份预设，验证成立。

**差异化验收（2026-07-20，用户通过）**：水边 v2 与旷野拉开差距的手段——① 音频多层合成（LFO 调频率=风的呼啸 / 调音量=浪的节奏，三层：湖鸣+拍岸 0.33Hz+粼粼细响）；② 地形按高度色带 ramp（水下泥→湿岸深绿→草坡翠绿）；③ 真蓝天+白云（沙漠天空刻意空，水边反其道）；④ 引擎新原子：ramp / clouds / bushes / 多树+dest 标记。**教训：新生态必须换配方结构，不能只改参数值，否则"沙漠里的水边"。**

**马赛克墙原型（2026-07-20）**：`wall/index.html`——1189 格全量（66 卷按十大分组排布），lvl 0-5 视觉演进（未亮→初亮→观察→明白→联结→活出），点击格子开详情面板（升级/查看联结/关闭），悬浮提示，进度 localStorage 持久化。

**串珠玩法（2026-07-21）**：确立"平台不创作解释"原则——我们非神职，桌上只放确凿之物（经文原文/TSK 串珠事实/读者 UGC）。核心循环="你也是串珠人"：读 A 短段→共词发现→读 B 短段→划锚写理由（先写后看）→「造就了我」→点亮连线。解读唯一合法形态=公版解经家引用+署名（策展不创作）。详见《串珠玩法设计.md》。
**产品定名「串珠」+三大铁则（2026-07-21）**：叙事根基=沙漠修道院的祷告与手工（沙漠教父默祷时编篮子）。铁则：① 无干扰项无选择题无对错——确定的珠子交给用户串，一次一串，可换一串/看答案；② 高级用户可新建串珠，社区审议后入"社区集"（与 TSK 集永远分层，定期 CC-BY-SA 回馈开源）；③ LLM 只做翻译（理由互译/公版注释中译），不碰解释。

**阶段划分（2026-07-21 定）**：注册与自创珠**推迟到下阶段**。本阶段=单机版收尾（打磨→CloudBase 部署→伙伴反馈），localStorage 存档、零后端。下阶段=社区版（注册→云端同步→众人注脚/造就了我→串珠人等级→自创珠+审议+翻案），需后端选型；衔接设计：本机存档一键并入账号，种子用户积累不丢。理由：社区功能需真实用户反馈打底；自创珠按规矩需先串够量（先临摹再创作），顺序天然。

**境外主体决策（2026-07-21 定）**：社区与服务器全部境外，境内牌照不申请。含义：主用户群=海外华人教会/团契/国际用户（大陆可用是加分非承诺，被墙风险接受）；现 CloudBase 静态站降级为过渡期国内镜像非长期资产；落地=Cloudflare Pages 主站（用户自注册账号+域名 ~$10/年）→ deploy 脚本双目标 → 社区版直接建在海外栈（Supabase 等）。与多语言路线互证：串珠主场在海外。

**开销估算（2026-07-21，约数）**：P1 静态版全年 ≈¥150-250（域名为主，CloudBase 月付个位数，CF Pages $0）；P2 社区版起步 $0/月（Supabase 免费档）→ 认真运营 $25-45/月 → 万人 $75-170/月（第一年 ≈¥2200-4400）。结构：静态内容永远近乎免费，成本只随"用户独有数据"增长；LLM 翻译千条注脚 <$1；最大开销是审核/策展人力非现金；免费+奉献制下几十个月奉献用户即可覆盖万人规模成本——财务可永续，无需融资。

**经营路线备忘（2026-07-21）**：① 海外=双部署非加服务器（静态托管 Cloudflare Pages $0，域名 $10/年；deploy 加一目标即可）；② 社区版后端 BaaS：海外 Supabase $0-25/月，国内 CloudBase ¥50-200/月——**但境内公开宗教 UGC 需互联网宗教信息服务许可证（个人几乎不可得），社区版现实路径=海外主体运营，境内保持无注册静态工具双轨**；③ OPC 方法论（slavingia/skills，《极简创业者》10 技能）适配度高——已走完找社群/验证/MVP，下两步跑 first-customers 与 pricing（免费+奉献制伦理待议）；processize 洞见：**伙伴微信群=众人注脚的手工版**，先人工验证"有人愿为珠写理由"再写社区代码。
